    jq getting interesting
    2020-05-07 15:21

    Where were we?

    Last time, we created a striing from each of the outputs we were interested in using a combination of the pipe ( | ), the raw and compact flag and the escaped round braces:

    jq -cr '.Reservations[].Instances[] | "\(.InstanceId): \(.State.Name): \(.PrivateIpAddress)"'

    Perhaps we would like to create an file array of the values. This is pretty simple because, in effect what we’re doing on the right hand side of the pipe ( | ) is to create a new structure. That structure itself could be a string like that above, or it could be an array

    jq -cr '.Reservations[].Instances[] | [.InstanceId), .State.Name, .PrivateIpAddress]'

    Not when you do this, it seems to ignore the raw ( -r ) flag. That may seem strange, but in fact creating an array in JSON means that elements of the array HAVE to use double quotes ( “ ) and must be separated by commas ( , ). That’s JSON.

    ["i-053667d4e26795541","running","172.16.2.26"]
["i-0a775b8c631e8ad27","running","172.16.2.15"]
["i-06e986e85261c838e","stopped","172.16.1.10"]
["i-061b358b9cfc20505","running","172.16.1.13"]

    So how about creating a new object in JSON with each element taggeg with a key?

    jq -cr '.Reservations[].Instances[] | {instance:.InstanceId), state:.State.Name, ip:.PrivateIpAddress}'

    produces

    {"instance":"i-053667d4e26795541","state":"running","ip":"172.16.2.26"}
{"instance":"i-0a775b8c631e8ad27","state":"running","ip":"172.16.2.15"}
{"instance":"i-06e986e85261c838e","state":"stopped","ip":"172.16.1.10"}
{"instance":"i-061b358b9cfc20505","state":"running","ip":"172.16.1.13"}

    NOTE: Worth noting here is that if I leave out a comma ( , ) between each element, this produces an error because that’s invalid JSON.

    jq: error: syntax error, unexpected IDENT, expecting '}' (Unix shell quoting issues?) at <top-level>, line 1:
.Reservations[].Instances[]|  {instance:.InstanceId,     state:.State.Name ip:.PrivateIpAddress}                                                                           
jq: 1 compile error

    The reason for this error was I left out a comma betwwen the state:.State.Name and the ip.PrivateIPAddress. That results in an error in the newly created JSON.

    What’s cool about this is that, say I would also like to know the state of block storage on the instances that are running

    jq -cr '.Reservations[].Instances[] | [.InstanceId), .State.Name, .PrivateIpAddress, .BlockDeviceMappings]'

    Lesson 7

    Selecting only running instances

    I certainly would like to know what instances are running, filtering out (discarding) those that are stopped.

    Select is the answer, with a note or two

    jq -c '.Reservations[].Instances[] | select(.State.Name == "running") | [.LaunchTime, .InstanceId, .State.Name, .PrivateIpAddress]'

    Just like in a Unix pipe, one can use select to select what you’re after and once you have it, you pipe those results to the output just as before.

    The command above produces only the running instances:

    ["2020-05-07T03:45:33.000Z","i-053667d4e26795541","running","172.16.2.26"]
["2020-05-07T03:50:45.000Z","i-0a775b8c631e8ad27","running","172.16.2.15"]
["2020-05-07T03:50:45.000Z","i-061b358b9cfc20505","running","172.16.1.13"]

    NOTE: I know that for this customer, instances are started every morning, so adding LaunchTime as the first element of the array shows me that indeed, these instances were launched today.
